File paths

When loading modules with require the path can be either relative to current script or relative to main script. Current script means the script that is loading the module. Main script means the script that was used to start the game. Filepaths relative to current script is prefixed with '/', without a leading '/' the filepath will be relative to the main script.

The same principal applies when using other filepaths (e.g. when creating a texture or shader). Note that the current script is the script executing the code that is using the filepath, not necessarily the script where the filepath itself exists. Sometimes it's needed to get the path of the current module, in those cases module.path can be used.

The contents of game.js

// Load module relative to main script
var ship = require('objects/ship.js');

The contents of objects/ship.js

// Filepath will be objects/assets/ship.png
var filepath = module.path + 'assets/ship.png';

Starting the play runtime with game.js as main script.

% play game.js
