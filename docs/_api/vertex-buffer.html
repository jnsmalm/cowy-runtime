VertexBuffer

Stores vertex data, used when drawing primitives.

Vertex declaration

The vertex declaration describes the attributes for the vertex data. It's an array with objects that has attributeName and attributeType properties. The vertex declaration must match the shader program used. For example, the vertex shader below has the following in-parameters.

in vec3 position;
in vec4 color;
in vec2 textureCoords;

The vertex declaration for that shader should have the attributes below, the order is important when setting the vertex data. The attribute type can have any of the following values: 'float', 'vec2', 'vec3', 'vec4'. 

var vertexDeclaration = [
  { attributeName: 'position', attributeType: 'vec3' },
  { attributeName: 'color', attributeType: 'vec4' },
  { attributeName: 'textureCoords', attributeType: 'vec2' }
];

Vertex data

The vertex data is an array with values to store in the vertex buffer. The order of the vertex data must match the vertex declaration. When setting data for the vertex buffer a hint is specified for how the buffer data will be stored. If the data is modified very few times it should be set to 'static'. If the data is modified many times it should be set to 'dynamic'.

// Create the vertex declaration.
var vertexDeclaration = [
  // (x, y, z)
  { attributeName: 'position', attributeType: 'vec3' },
  // (u, v)
  { attributeName: 'textureCoords', attributeType: 'vec2' }
];
 
// var window = new Window(...)
var vertexBuffer = new VertexBuffer(window.graphics, vertexDeclaration);
 
// Array with vertices that matches the declaration above (x, y, z, u, v).
var vertices = [
   // Vertex 1
   0.0,  0.5, 0.0, 0.0, 0.0,
   // Vertex 2
   0.5, -0.5, 0.0, 0.0, 0.0,
   // Vertex 3
  -0.5, -0.5, 0.0, 0.0, 0.0,
];
 
// The usage can be set to 'static' or 'dynamic'.
var usage = 'dynamic';
 
// Sets the vertex data for the buffer.
vertexBuffer.setData(vertices, usage);
